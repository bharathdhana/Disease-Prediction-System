# Disease Prediction System Documentation

## 1. Project Overview
The **Disease Prediction System** is a full-stack web application designed to predict the likelihood of various diseases based on user-provided symptoms and medical data. It utilizes Machine Learning models for accurate predictions and provides a user-friendly interface for interaction.

The system currently supports prediction for:
- Heart Disease
- Diabetes
- Parkinson's Disease


## 2. Features

### Disease Prediction Modules
*   **Heart Disease Prediction**: Analyzes cardiovascular metrics using an ensemble of **Deep Learning (CNN & ANN)** models.
*   **Diabetes Prediction**: Uses a **Deep Neural Network (DNN)** to predict diabetes risk based on medical indicators.
*   **Parkinson's Prediction**: specific **Convolutional Neural Network (CNN)** for vocal feature analysis.


### Key Capabilities
*   **Web Interface**: Intuitive HTML/CSS frontend for data entry.
*   **Real-time Analysis**: Instant results generated by trained ML models.
*   **Confidence Scores**: Provides a probability percentage for each prediction.
*   **History Tracking**: Logs all predictions to a local SQLite database for review.
*   **Visualizations**: Displays performance metrics for the models.

## 3. Technical Architecture

### Technology Stack
*   **Frontend**: HTML5, CSS3, JavaScript (Flask Templates).
*   **Backend**: Python, Flask Web Framework.
*   **Database**: MySQL (`disease_prediction`).
*   **Deep Learning**:
    *   `TensorFlow` / `Keras` for all models (ANNs, CNNs).
    *   `numpy` & `pandas` for data manipulation.

### Utility Tools
*   **Database Viewer (`view_db.py`)**: A desktop GUI application (Tkinter) to view the prediction history.
    *   **Features**: tabular view of records, "Refresh Data" button, and "Clear History" button to wipe the database.
    *   **Usage**: Run `python view_db.py`.
*   **Testing Guide (`TESTING_GUIDE.md`)**: Contains 5+ test cases for each disease model to verify predictions.

## 4. Database Design
The application uses a single table `predictions` in MySQL to store history.

**Table Schema (`predictions`):**
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | INTEGER PK | Auto-incrementing unique ID |
| `disease_type` | TEXT | Type of prediction (e.g., "Heart Disease") |
| `input_features` | TEXT | JSON string of input values used |
| `prediction_result` | TEXT | The outcome (e.g., "Detected", "Not Detected") |
| `probability` | REAL | Confidence score of the prediction |
| `timestamp` | DATETIME | Time of prediction (Auto-generated) |

## 5. Installation & Setup

### Prerequisites
*   Python 3.8 or higher
*   pip (Python package manager)

### Installation Steps

1.  **Clone/Open the Project**
    Navigate to the project directory:
    ```bash
    cd d:/code3/disease_prediction
    ```

2.  **Create a Virtual Environment (Recommended)**
    ```bash
    python -m venv .venv
    .\.venv\Scripts\activate
    ```

3.  **Install Dependencies**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Run the Application**
    You can run the application using the included batch file or via Python directly:
    ```bash
    # Option A: Using the batch file
    run_app.bat

    # Option B: Direct Python execution
    python app.py
    ```

5.  **Access the App**
    Open your browser and visit: `http://127.0.0.1:5000`

## 6. How to Use
1.  **Home Page**: Select the disease type you want to predict from the dashboard.
2.  **Input Form**: Fill in the required medical values (inputs are validated to be numbers).
3.  **Result**: View the prediction outcome, confidence score, and model metrics.
4.  **History**: Navigate to the "History" page to see a log of past predictions.
